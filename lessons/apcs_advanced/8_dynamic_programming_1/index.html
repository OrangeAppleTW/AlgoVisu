動態規劃（一）

透過爬樓梯問題的視覺化演示，比較遞迴與動態規劃的效率差異，學習動態規劃的核心概念

什麼是動態規劃？

動態規劃（Dynamic Programming）是一種演算法設計技巧，核心思想是「避免重複計算」。 當我們用遞迴解決問題時，常常會重複計算相同的子問題，動態規劃透過記憶化（Memoization） 或表格法（Tabulation）來儲存已計算的結果，大幅提升效率。

以爬樓梯問題為例：到達第 n 階的方法數等於到達第 (n-1) 階和第 (n-2) 階的方法數總和。 遞迴解法會重複計算許多子問題，而動態規劃只計算一次，時間複雜度從 O(2^n) 降為 O(n)。

兩種方法的關鍵差異

遞迴解法

思路: 由上而下分解問題

複雜度: O(2^n) 指數時間

問題: 大量重複計算

空間: O(n) 遞迴堆疊深度

動態規劃

思路: 由下而上建構解答

複雜度: O(n) 線性時間

優勢: 每個子問題只算一次

空間: O(n) 儲存中間結果

效率對比

f(10): 遞迴 177 次 vs DP 10 次

f(20): 遞迴 21891 次 vs DP 20 次

f(30): 遞迴 2.7M 次 vs DP 30 次

實際應用: DP 可解更大規模問題

🎯 教學流程建議

先用遞迴解法讓學生理解問題結構，再引入動態規劃優化，強調記憶化的重要性。

教學重點：
  • 對比兩種解法的計算過程
  • 強調重複子問題的識別
  • 展示記憶化的實際效果
  • 分析時間複雜度的差異

💡 學習要點

動態規劃適用於具有重複子問題和最佳子結構的問題，是演算法優化的重要技巧。

核心概念：
  • 重複子問題（Overlapping Subproblems）
  • 最佳子結構（Optimal Substructure）
  • 記憶化技巧（Memoization）
  • 狀態轉移方程（State Transition）
